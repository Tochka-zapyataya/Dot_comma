# Текущий лучший результат

**v14** — avg WAPE 10.141% по 3 holdout-окнам (апр 8.941%, фев 10.298%, окт 11.185%)

# Что работает (v14)

- ETS(sp=112) + XGBoost(MAE) + LightGBM(MAE) + Analog Ensemble
- Holiday alpha boost: alpha_analog=0.85 для праздников с historical pool
- Holiday YoY cap: hol_yoy = min(YOY_TREND, 1.0)
- Volume-weighted sample weights: пиковые часы (12-19) больший вес
- slot_ewm12: 12-недельный EWM по слоту
- Blend XGB/LGB: 0.5/0.5
- **Morning downweight**: часы 7-10 получают vol_w × 0.3 при обучении (NEW v14)
- No-leakage weather: known_cutoff в fetch_weather_api

## Критический паттерн во всех провалах

Часы 12-13 (пиковые, наибольший вес в WAPE) **стабильно ухудшались** при любом изменении:
- v14a: +0.46%, +0.67%
- v14b: +0.18%, +0.42%
- v14c: +0.92%, +1.34%

При этом часы 15-18 стабильно улучшались. Это значит v13 уже оптимален для пиковых часов — любое "улучшение" нарушает их калибровку.

## Причины провала каждой попытки

- **OOF blend**: нашёл XGB=0.95/LGB=0.05 — оверфит на CV-фолдах
- **CatBoost**: CV лучше (7.644% vs 7.667%), но holdout хуже — признак переподгонки. CAT имеет 2117 итераций vs XGB 1451 — возможно, переобучился на residuals часов 12-13
- **doy features**: незначительно сдвигают прогноз в неправильную сторону для конца апреля
- **No-leakage weather**: корректная практика, на WAPE почти не влияет (погода апреля предсказуема)
